# WebAssembly Build Process

This document outlines the steps to compile the C++ project into a WebAssembly (Wasm) module, allowing it to run in a web browser.

## Prerequisites

Before you can build the project for WebAssembly, you must have the [Emscripten SDK](https://emscripten.org/docs/getting_started/downloads.html) installed and activated in your shell. For initial setup instructions, refer to the `wasm-plan.md` document in this directory.

## Build Script: `build-wasm.sh`

The project includes a shell script, `build-wasm.sh`, which automates the compilation process. The script performs the following actions:

1.  **Creates a `wasm` directory:** All the generated files for the web build will be placed in `starter-quest/wasm`.
2.  **Compiles C++ source:** It gathers all the necessary `.cpp` source files from the `baseClasses`, `characterClasses`, and `utils` directories.
3.  **Invokes Emscripten (`emcc`):** It uses the Emscripten compiler to build the C++ files into a JavaScript file (`quest.js`) and a WebAssembly module (`quest.wasm`).
4.  **Copies `index.html`:** It copies the custom `starter-quest/index.html` file into the `wasm` directory to provide a user interface for the application.

## How to Build and Run

Follow these steps to build and run the application in your browser:

1.  **Make the build script executable:**
    If you haven't done so already, you need to give the script execute permissions.

    ```bash
    chmod +x starter-quest/build-wasm.sh
    ```

2.  **Run the build script:**
    Execute the script from the root of the project.

    ```bash
    ./starter-quest/build-wasm.sh
    ```
    This will compile the project and create the `wasm` directory with all the necessary files.

3.  **Start a local web server:**
    WebAssembly modules need to be served over HTTP(S). Navigate to the output directory and start a simple Python web server.

    ```bash
    cd starter-quest/wasm
    python3 -m http.server
    ```

4.  **Open in browser:**
    Once the server is running, open your web browser and navigate to:

    [http://localhost:8000](http://localhost:8000)

You should see the "Starter Quest" application running, with its output displayed in a text area on the page.

## Key Files

-   `build-wasm.sh`: The main build script for compiling to WebAssembly.
-   `index.html`: The custom HTML file that provides the user interface for the web application.
-   `wasm/quest.js`: The JavaScript "glue" code generated by Emscripten to load and run the WebAssembly module.
-   `wasm/quest.wasm`: The compiled WebAssembly binary containing the C++ application logic.
